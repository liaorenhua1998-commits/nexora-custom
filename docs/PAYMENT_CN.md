## 支持的支付方式

| 服务商 | 支付方式 | 说明 |
| --- | --- | --- |
| EasyPay | 支付宝、微信支付 | 通过 EasyPay 协议接入第三方聚合支付 |
| 支付宝直连 | 桌面端二维码、移动端跳转 | 直接接入支付宝开放平台 |
| 微信支付直连 | Native、H5、JSAPI | 直接接入微信支付 APIv3 |
| Stripe | 银行卡、支付宝、微信支付、Link 等 | 适合国际支付与多币种结算 |

> 前台固定只展示 `支付宝` 和 `微信支付` 两个可见入口。后台可以为这两个入口分别指定直连通道或 EasyPay 通道。

---

## 快速开始

1. 进入管理后台的 **设置 -> 支付设置**
2. 打开 **启用支付**
3. 配置金额范围、超时时间、待支付订单上限等基础参数
4. 在 **服务商管理** 中添加至少一个支付服务商实例
5. 用户即可在前台发起充值

## 系统设置

### 基础设置

| 设置项 | 说明 | 默认值 |
| --- | --- | --- |
| 启用支付 | 开启或关闭整套支付能力 | 关闭 |
| 商品名前缀 | 显示在支付页的前缀 | 空 |
| 商品名后缀 | 例如“积分”“额度” | 空 |
| 最低金额 | 单笔最小充值金额 | 1 |
| 最高金额 | 单笔最大充值金额，留空表示不限 | 空 |
| 每日限额 | 单用户每日累计充值上限 | 空 |
| 订单超时 | 待支付订单超时分钟数 | 30 |
| 最大待支付订单数 | 同一用户并行待支付订单上限 | 3 |
| 负载均衡策略 | 多实例服务商的分配策略 | 轮询 |

### 前台支付方式路由

- 支付宝入口可路由到 `支付宝直连` 或 `EasyPay 支付宝`
- 微信支付入口可路由到 `微信支付直连` 或 `EasyPay 微信`
- 同一入口同一时刻只能绑定一个来源
- 如果只启用了入口但没有指定来源，前台不会显示该支付方式

## Webhook 配置

系统会根据站点域名生成回调地址。典型路径如下：

| 服务商 | 回调路径 |
| --- | --- |
| EasyPay | `/api/v1/payment/webhook/easypay` |
| 支付宝直连 | `/api/v1/payment/webhook/alipay` |
| 微信支付直连 | `/api/v1/payment/webhook/wxpay` |
| Stripe | `/api/v1/payment/webhook/stripe` |

生产环境建议：

- 使用 HTTPS
- 确保反向代理放行 `/api/v1/payment/webhook/*`
- 支付成功与充值成功分开落库，便于失败重试

## 支付流程

1. 用户选择充值金额和支付方式
2. 系统校验金额范围、待支付订单数和每日限额
3. 系统按策略选择一个可用的服务商实例
4. 创建订单并向上游服务商发起支付
5. 用户完成支付
6. 平台验签并处理 webhook
7. 订单状态更新为已支付，再完成余额充值

常见订单状态：

- `PENDING`：待支付
- `PAID`：已支付，待充值
- `COMPLETED`：充值完成
- `EXPIRED`：超时未支付
- `CANCELLED`：用户取消
- `FAILED`：充值失败，可重试
- `REFUND_REQUESTED` / `REFUNDING` / `REFUNDED`：退款流程相关状态

## 从外部支付桥迁移

如果你此前使用外部支付桥，现在可以迁移到 Nexora 内置支付。

### 主要差异

| 对比项 | 外部支付桥 | 内置支付 |
| --- | --- | --- |
| 部署方式 | 独立服务 | 内置在 Nexora 中 |
| 配置方式 | 环境变量 + 独立后台 | Nexora 管理后台统一配置 |
| 订单管理 | 独立界面 | 集成在 Nexora 管理后台 |
| 回调处理 | 通过外部桥回调到业务系统 | 平台内部直接处理 |

### 迁移步骤

1. 在 Nexora 管理后台启用支付
2. 使用原有支付凭证配置新的支付服务商
3. 将各支付服务商 webhook 改为 Nexora 回调地址
4. 用测试订单验证支付和充值链路
5. 确认无误后下线旧支付桥

> 历史订单不会自动迁移。若仍需查询旧订单，请保留旧系统一段时间。
